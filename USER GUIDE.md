The DMM Bronx Auto System is a web-based car rental management application developed using PHP and MySQL. It allows users to manage cars, customers, and rental records through an easy-to-use interface. The system connects to a MySQL database and performs operations such as inserting, viewing, and deleting records using SQL queries and joins. The database connection is handled through a configuration file that connects to the local MySQL server using the database name dmmbronxauto, ensuring that all pages can access and manipulate the stored data properly.

To begin using the system, the user must first start XAMPP and ensure that both Apache and MySQL services are running. After that, the database must be imported into phpMyAdmin. This is done by opening a browser and navigating to dmmbronxauto, and importing the provided SQL file. This file creates all required tables, including cars, customers, rentals, cartypes, and payments, along with their relationships and sample data. Once the database is successfully imported, the system can be accessed by going to http://localhost/dmmbronxauto/ in the browser.

When the system loads, the user is taken to the homepage, which serves as the main navigation hub. The homepage provides a welcome message and displays sections that allow the user to navigate to cars, customers, and rentals. It also includes a navigation bar at the top of the page that provides direct access to all main features of the system, including viewing cars, viewing customers, viewing rentals, adding a customer, and adding a rental. This page is designed to give users quick access to all parts of the application and is powered by the main PHP file responsible for loading the system interface.

To view car records, the user can click on the “View Cars” option from the navigation bar. This page displays all cars stored in the database in a table format, including details such as car ID, brand, model, year, type, and availability. The system uses a SQL JOIN operation to combine data from the cars and cartypes tables, allowing the user to see the type name associated with each car instead of just an ID. The availability field is also displayed in a user-friendly format, showing whether a car is available or not.

To view customer records, the user can click on “View Customers.” This page displays all customers in a table, including their customer ID, first name, last name, email, and phone number. Each customer record also includes a delete option, which allows the user to remove a customer from the system. When the delete link is clicked, the system sends the customer ID to a separate PHP file that securely deletes the record using a prepared statement and then redirects the user back to the customers page.

To add a new customer, the user can navigate to the “Add Customer” page. This page contains a form where the user must enter the customer’s first name, last name, email, and phone number. All fields are required, and the system performs validation to ensure that the email is in a valid format and that the phone number contains only digits between 10 and 15 characters. Once the form is submitted, the data is securely inserted into the customers table using a prepared SQL statement. A success or error message is then displayed to inform the user of the result.

To create a rental, the user can go to the “Add Rental” page. This page allows the user to select an existing customer and car from dropdown menus, which are automatically populated from the database. The user must then enter a rental date, return date, and select a status such as Booked, Active, or Returned. The system validates that all fields are filled and ensures that the return date is not earlier than the rental date. Once submitted, the rental is added to the database using a prepared statement, and a confirmation message is displayed.

To view rental records, the user can click on “View Rentals.” This page displays all rental transactions, including the customer’s name, the car rented, rental date, return date, and status. The system uses SQL JOIN operations to combine data from the rentals, customers, and cars tables, allowing all relevant information to be displayed together in one table. This makes it easier for users to understand the relationship between customers and their rented vehicles.

There are several important notes when using the system. All input fields must be filled out before submitting forms. Email addresses must be valid, and phone numbers must contain only numeric values between 10 and 15 digits. When creating rentals, the return date must always be later than the rental date. The system also uses prepared statements throughout to improve security and prevent SQL injection.
